从递归到动规题目（9）

Coffee

题目描述

给定一个数组arr，arr[i]代表第i号咖啡机泡一杯咖啡的时间。给定一个正数N，表示N个人等着咖啡机泡咖啡，每台咖啡机只能轮流泡咖啡。只有一台咖啡机，一次只能洗一个杯子，时间耗费a，洗完才能洗下一杯。每个咖啡杯也可以自己挥发干净，时间耗费b，咖啡杯可以并行挥发。假设所有人拿到咖啡之后立刻喝干净，请返回从开始等到所有咖啡机变干净的最短时间。（三个参数：int[] arr、int N，int a、int b）。

算法分析

题目主干两部分的分析

  • 通过审题可知，本题分为两个部分，第一是N个人去泡咖啡的过程，第二是N个人洗咖啡的过程。我们现在分别对这两个过程进行分析。
  1. 泡咖啡过程分析:
    • 分析可知，这个过程要求我们做的，其实就是寻找出一个最优的排队方案，让所有人喝完之后到达的时间点最小。arr的长度即泡咖啡机的个数，可以想像N个人同时在0号时间点涌向这写泡咖啡的机器，进行排队。由于每一个泡咖啡的机器的泡咖啡速度是不一样的，那我们就必须给这N个人一个决策。我们站在上帝视角去分配一种最优的排队方案，让泡咖啡的效率最高，即让N个人都泡完咖啡的时间点最小，我们可以返回一个长度为N的数组，代表着每一个人泡完咖啡的时间点（递增，有序），而这个数组必须是最优的。
    • 我们现在思考一下如何返回这个最优数组。宏观上来看，这道题是一个样本数量在变的、想求最大最小的问题，很容易想到用堆。我们可以维护一个小根堆，那么就很自然牵扯到排序标准，什么样的情况才能说“小”。经过思考可以知道，判断堆元素大小的指标因该是：该泡咖啡的机器泡一杯所需的时间加上该咖啡机下次能被使用的最近的时间点。所以我们可以用一个类去封装一下这两个数据，一个是最近能被使用的时间点，另一个就是冲咖啡所要用的时间；
    • 我们来讨论一下，这个和意味着什么。显然，这就是下一个人能用的时间点。也就是我们的决策就是，按照下一个人能用的时间点从小到大进入小根堆即可！

  2. 将咖啡杯弄干净的过程：
  • 首先我们在这之前已经得到了N个人喝完咖啡的时间节点来到哪里。对于每一个人，都有两种选择，一是选择洗，而是选择不洗。
  • 建立一个递归过程：int process(int[] donePoint, int index, int a, int b, int availableTimePoint)表示： 现在来到index位置的人做选择，假设[0...index-1]的人已经做好选择，咖啡机再次可用的时间是availableTimePoint，返回全部解决的最小时间节点；
  • 我们主要来考虑一下baseCase：
    • 这种情况下返回什么是很有讲究的
    • 为什么要返回0呢？我们可以理解成，这一步到头了，对前面的决策没有影响，做决策的数据主要是和donePoint数组中的值有关，在basecase的时候没必要对数据进行操作，所以直接返回即可。
    • 如果想理解本质，我们可以画一张图去解释。
    • 由上图可以知道，f(6)向f(5)返回什么其实取决于怎么样才能让f(5)自洽地得到答案。很显然，f(5)调用f(6)是为了去进行max操作的，所以返回0显然是一个正确的选择。
  • 这个递归的其他过程就顺其自然出来了。

对于递归过程的优化

  • 这要弄成动态规划，其实简单的方法是弄成记忆化搜索，因为时间复杂度来看的话，记忆化搜索和弄成结构化dp的代价差不多。弄成结构化dp的一大难点是，很难知道availableTimePoint这个变量的取值范围。所以我们试着分析一下这个变量的范围
  • 首先这个变量的最小值是0，最大值的话，是所有咖啡杯都用来洗，最终的时间点。
  • 首先进行简单进行数学分析，洗一杯的时间是a，一共有N个杯子，所以最大值是a*N
  • 上述的想法看似正确其实有漏洞。很显然，并不一定一个杯子洗完之后，马上就能洗另外一个杯子，很有可能还没喝完。
  • 所以我们需要用循环去求出这个最大值
  • 求出最大值之后，一切都那么丝滑。
