从递归到动态规划_Problem1

RobotWalk

题目描述

假设有排成一行的N个位置记为1～N，N一定大于或等于2，开始时机器人在其中的M位置上(M一定是1～N中的一个)。如果机器人来到1位置，那么下一步只能往右来到2位置；如果机器人来到N位置，那么下一步只能往左来到N-1位置；如果机器人来到中间位置，那么下一步可以往左走或者往右走。现规定机器人必须走K步，求机器人最终能来到P位置（P也是1~N中的一个）的方法有多少种。给定四个参数 N、M、K、P，返回方法数。

算法分析

这道题是暴力递归到动态规划的第一题，具有很强的启发意义。下面从暴力递归和动态规划两方面来实现这道题。难度属于简单。

解法1——暴力递归

很显然定义一个递归过程，int process(int cur, int rest, int aim, int N)，这个递归的含义如下定义：机器人当前站在cur位置，还有rest步要走，目标位置是aim，位置编号是1-N。很容易写出如下递归：

简单来说，就是分四大种情况：

  1. 如果目前没有步数可以走了，判断是否目前到达终点，是则返回1，不是则返回0；
  2. 如果目前在1位置，只需尝试下一步在2位置的总方法数；
  3. 如果目前在N位置，只需尝试下一步在N - 1位置的总方法数；
  4. 如果上述条件都不满足，即目前在1-N中间的某个位置cur，需要计算出下一步在cur - 1位置和下一步在cur + 1位置分别的方法数之和，即为当前在cur位置的方法数。

这样就可以尝试完所有的情况，把结果返回即可。

解法二——记忆化搜索

记忆化搜索就是搜索的时候用一张dp表，注意到在这个递归过程中，变量只有两个，每一对变量对应一个状态，且这个状态的答案只是与状态参量（两个变量）有关，和前后过程没有关系。所以，当这两个变量确定了，答案也就自然确定了。

（1）初始化时，先让所有的dp[i][j]都变成-1，表示这个状态没有算过。算过了就更改值为当前状态的答案。

（2）递归过程中，我们首先判断当前进入的递归有没有被算过：

  1. 如果被算过，直接返回dp表里当前状态的答案；
  2. 如果没有被算过，首先我们要用递归找到当前答案，填补到dp表当前的状态上，最后返回即可。
  3. 如何找到当前答案呢？逻辑和纯暴力搜索是一样的，直接复制过来即可。

解法三——结构化的dp

  • 从递归改结构化的dp主要有这么几个步骤：
  1. 分析变量的个数，分析出dp表的维数；
  2. 分析出dp表每一个维数的数据范围，如果到无限的话，需要考虑其他尝试过程；
  3. 根据dp表的数据范围确定每一维的个数；
  4. 用不到的位置要了然于心；
  5. 根据basecase去填表的基本数据（初始化dp表）；
  6. 分析表中任意一个位置的依赖（运用尝试过程去搞）；
  7. 根据依赖过程分析出填写表格的大方向和小方向（这边技巧特别多，一般是用for循环的各种模拟即可）；
  8. 写出循环大逻辑；
  9. 根据尝试过程填表。
