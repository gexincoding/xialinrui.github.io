从递归到动规题目（8）

HorseJump

题目描述

请自行搜索或者想象一个象棋的棋盘，然后把整个棋盘放入第一象限，棋盘的最左下角是(0,0)位置那么整个棋盘就是横坐标上9条线、纵坐标上10条线的区域。给你三个 参数 x，y，k。返回：“马”从(0,0)位置出发，必须走k步，最后落在(x,y)上的方法种数。

算法分析

暴力递归

  • 想法是：设计一个递归函数，int process(int x, int y, int a, int b, int rest)，表示，我现在在(x,y)，我要去(a,b)，剩余rest步，返回方法数。

    • 到达每一个点，我都有8种选择。

    • 只要将这八种选择所得到的结果累加返回即可。

  • 如果当前越界了，给上游返回0，有两层意义：

    • 第一就是，在上游来看，我这种尝试是错误的
    • 第二就是，在上游来看，累加的时候，加的是0，相当于没有算。

动态规划

  • 分析变量：

    • x，y，rest是变量；
    • x的变化范围是0-8；y的范围是0-9；rest的范围是0-k；
    • x的空间是9，y的空间是10，rest的空间是k+1；

  • 如何排除无效解？

    • 如果当前参数越界了，返回0
    • 否则的话，就返回格子里的值即可！

  • dp表的填写顺序和依赖关系

    • 上层只依赖下层
    • 从一层之间的填表顺序没什么要求
