Problem002_AddTwoNumbers

题目描述

给你两个非空的链表，表示两个非负的整数。它们每位数字都是按照逆序的方式存储的，并且每个节点只能存储一位数字。

请你将两个数相加，并以相同形式返回一个表示和的链表。

你可以假设除了数字 $0$ 之外，这两个数都不会以 $0$ 开头。

  • 示例 1：

    输入：`l1 = [2,4,3], l2 = [5,6,4]` 输出：`[7,0,8]` 解释：`342 + 465 = 807`.

  • 示例 2：

输入：l1 = [0], l2 = [0]
输出：[0]

  • 示例 3：

输入：l1 = [9,9,9,9,9,9,9], l2 = [9,9,9,9]
输出：[8,9,9,9,0,0,0,1]

  • 提示：

每个链表中的节点数在范围 [1, 100] 内
$0 <= Node.val <= 9$
题目数据保证列表表示的数字不含前导零

算法分析

自我分析

  • 本道题很简单，简单模拟一下进位，考察的点是链表基本操作——尾插法。

  • 不过，在自己写代码时候，之前一直没有过，后来发现了一个小问题，是通过暴力打印结果的方式发现的，还是很有收获。

  • 先简单说一下流程：

  1. 首先建立一个存储答案的链表首节点，不存任何值，返回的时候直接返回next即可；
  2. 用p1、p2分别记录两个链表当前遍历到的节点是什么，初始化为l1和l2的头节点；
  3. 用一个整形变量add记录当前进位，每一次遍历之后进行修改，传递到下游。也就是说下游收到的add就是本次加分需要的进位，上游已经处理好了；
  4. 开始遍历，因为终止条件是：两个遍历的节点都为空，才结束；
  5. 进入这个循环，有两种大情况：p1和p2中有一个为空、两个都不为空。
  6. 第一种情况，以p1为空p2不为空为例，这一次的运算结果应该是p2的值加上进位；判断得出的计算结果大于等于还是小于10，根据不同情况传给下游的进位，接着将答案放进一个节点里，进行链表尾插。尾插过程用一个tail指针记录答案链表末尾的非空节点是什么，并且根据每次插入依次调整，这样就没必要每次尾插都重新从头遍历一遍答案链表了。最后移动非空p2指针；
  7. 第二种情况，就是运算的时候要变，其他操作一样；
  8. 如果跳出循环之后进位还不是0，说明最高位也有进位，需要重新开辟一个节点，进行连接；
  9. 让res链表后移一位，返回即可。

标答分析

  • 用数组

这是题解给出的一种做法嘛，就是很暴力地把链表中的数字放在ArrayList中，然后用高精度的那套思想，把答案也用一个数组收好。当然这种做法有点耍流氓，不考虑；

  • 先逆序，再倒叙，没啥必要 。

这是标答给的解法，和我一开始的分析没有什么区别，就是在每一位算出来的时候链表接的方式和我的接法是截然相反的，目的是省去先开一个头节点。但是其实又把链表遍历了一遍，时间复杂度常数项反而大了。
