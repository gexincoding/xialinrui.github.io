树状数组

树状数组能干啥

  • 树状数组实现的问题要分为一维、二维。现假设有一数组arr：
    • 一维树状数组：假设arr数组是一个一维数组。树状数组解决的问题是：
      1. 用log(N)的时间复杂度求出arr任意范围[L...R]上的累加和。由于累加和可以由前缀和数组简单求出，所以问题转移到了，这个结构能快速生成一个前缀和数组
      2. arr任意修改一个值，随之而来的调整代价也是极低的。（单点修改）
    • 二维树状数组：假设arr数组是一个二维数组。树状数组解决的问题是：
      1. 能够快速求出由arr[x1][y1]和arr[x2][y2]为对角线定点所围出的矩形区域中所有元素的累加和
      2. arr任意修改一个值，随之而来的调整代价也是极低的。（单点修改）

一维树状数组的实现

  • 本节所有数组下标的索引皆从1开始，0位置弃而不用，为的是后续位运算的合理性
  • 我们定义一个树状数组tree，结构实现如下：
    • 每一个位置填的数是原数组某一个区间上的累加和
    • 我们脑补一个概念，就是所要存的区间累加和的区间长度
    • 对于每一个位置而言，首先要记录自己到自己位置单个元素的值，要不要继续增加区间长度取决于当前已经存储的区间长度与之前位置存储的区间长度是否一致，一直的话就进行合并，不一致就不进行任何操作
    • 下面给一个具体的例子，假设原数组的长度是16
    • 注释：arr数组划横线代表当前所记录的最长区间长度
      • 首先初始化数组
      • 现在开始填写第一个位置
      • 第一个位置只能填自己到自己的累加和
      • 第一个位置之前没有填过，所以往后跳
      • 第二个位置首先填上自己
      • 但是发现，2位置所表示的区间长度是1，之前的区间长度填的也是1，相同，所以可以合并
      • 因此，2位置填的就是原数组1-2位置的累加和
      • 从3位置开始，以此按照这个规律
      • 最终能得到树状数组每一个位置填哪些区间的累加和
  • 认识了树状数组每个位置的填法怎么规定，我们现在要解决的就是如何去填。
    • 第一，每个位置如何快速知道所要填的区间的开始位置和末尾位置？
      • 结论：末尾位置是自己，开始位置是让自己最右侧的1变成0之后再+1的位置（二进制下）
    • 第二，如何求出原数组arr[index]位置的前缀和（从第一个位置一直加到index位置）?
      • 结论：我们只需在树状数组中找到一些特定的位置相加即可。这些位置是index不断把最右侧的1变成0之后的位置。
      • 举个例子，假设我们要求arr[1]一直加到arr[35]（也就是index = 35），首先我们要写出35的二进制形式是10011，所以我们要找的树状数组的位置是：10011位置、10010位置、10000位置，也就是这个前缀和等于tree[35] + tree[33] + tree[32]
      • 这种方式我们惊奇的发现，时间复杂度是O(1)
    • 第三，我现在对arr数组某一个index位置的值进行更新，我想快速知道tree树状数组怎么更新，也就是说我想知道改了这一个值之后，哪些位置会受到影响？
      • 结论是：这些受影响的位置是，从index不断加上最右侧的1及其后面的0所构成的数位置，最大位置当然不能超过数组长度。
      • 举个例子：假设我现在对arr中，index = 10011000位置的数进行更新。那么相应的，在tree中受到牵连（我们要更改数据保，以证结构正确性的）位置是：10011000，10011000 + 1000 = 10100000，10100000 + 100000 = 11000000，……位置

    二维树状数组的实现

  • 二维树状数组思想上和一维差不多，了解一下几个结论即可：
    • 第一，想算二维定义下的前缀和怎么算？
      • 结论：我们只需在树状数组中找到一些特定的位置相加即可。这些位置行和列都是相应的i和j不断把最右侧的1变成0之后的位置。
      • 举个例子，假设我们要求arr[2][3]的二维”前缀和”，首先我们要写出2的二进制形式是0010，所以我们要找的树状数组的横坐标是：0010位置；3的二进制是0011，所以我们要找的树状数组的纵坐标是0011位置、0010位置。这样算下来有三种组合，也就是说最终这个前缀和等于tree[2][2] + tree[2][3]
    • 第二，对arr中一个位置的值进行更新，tree哪些位置会受到影响而随之跟新？
      • 行受影响的位置是，从i不断加上最右侧的1及其后面的0所构成的数位置，最大位置当然不能超过行数；
      • 列受影响的位置是，从j不断加上最右侧的1及其后面的0所构成的数位置，最大位置当然不能超过列数；
      • 相互组合即可
