Problem1_CheckDistancesBetweenSameLetters

题目描述

给你一个下标从0开始的字符串s，该字符串仅由小写英文字母组成,s中的每个字母都恰好出现两次。另给你一个下标从0开始、长度为26的的整数数组distance。
字母表中的每个字母按从0到25依次编号（即，'a' -> 0, 'b' -> 1, 'c' -> 2, … , 'z' -> 25）。 在一个匀整字符串中，第i个字母的两次出现之间的字母数量是distance[i]。如果第i个字母没有在s中出现，那么distance[i]可以忽略 。 如果s是一个匀整字符串，返回true；否则，返回false。

Problem2_NumberOfWaysToReachAPositionAfterExactlyKStep

题目描述

给你两个正整数startPos和endPos。最初，你站在无限数轴上位置startPos处。在一步移动中，你可以向左或者向右移动一个位置。 给你一个正整数k，返回从startPos出发、恰好移动k步并到达endPos的 不同 方法数目。由于答案可能会很大，返回对1000000007取余的结果。 如果所执行移动的顺序不完全相同，则认为两种方法不同。 注意：数轴包含负整数。

示例 1：

示例 2：

提示：

  • 1 <= startPos, endPos, k <= 1000

算法思路

  • 看到方法数，直接用dp，由于是无限长，所以最好用记忆化搜索
  • 注意几个过滤条件：
    • 如果要走的步数比当前位置到终点的距离大，表示上游的决策让我现在无路可走，直接返回0
    • 距离和k的奇偶性必须相同，否则也是世界返回0
  • 进行这两步过滤直接打败百分之百
  • 当然这道题也可以用数学
  • 其实很简单，我们假设目前来到坐标为x
  • 假设我们向右走r，向左走k-r
  • 则有x+r-(k-r)=t
  • 则r=(k+t-x)/2
  • 记t-x=d,也就是距离
  • 也就是向右可以选择走1，2，3……一直到d步
  • 总的方法数是C(d,r)

Problem3_LongestNiceSubarray

题目描述

给你一个由正整数组成的数组nums。如果nums的子数组中位于不同位置的每对元素按位 与（AND）运算的结果等于0,则称该子数组为优雅子数组。返回最长的优雅子数组的长度。子数组是数组中的一个连续部分。
注意：长度为1的子数组始终视作优雅子数组。

示例 1：

示例 2：

提示：

  • 1 <= nums.length <= 1e5
  • 1 <= nums[i] <= 1e9

算法分析

  • 看到要算的是最长的字数组，这个字数组满足一定条件，很容易想到滑动窗口
  • 总体思路就是，满足条件l++，更新最大值；不满足条件r++，消除r位置对窗口内性质的影响
  • 我们的关注点就到了：满足的条件如何写以及如何消除r位置的影响
  • 分析可知，条件就是窗口内任意两个数的与运算要是0，那么如果我更新一个窗口就把所有任意两个数去判断一下。有没有更快捷的做法呢？
  • 其实是有的，我们用一个变量or去记录窗口中所有数进行或运算之后的值。当窗口往右扩的时候，我们只要比较一个新进的值与上or是不是0即可
  • 消除影响的时候，直接用or去异或l位置的值，影响就被消除了。
  • 时间复杂度为O(n)

Problem4_MeetingRoomsIII

题目描述

给你一个整数n，共有编号从 0 到 n - 1 的 n 个会议室。 给你一个二维整数数组 meetings ，其中meetings[i] = [starti, endi]表示一场会议将会在 半闭 时间区间 [starti, endi) 举办。所有 starti 的值 互不相同 。 会议将会按以下方式分配给会议室： 每场会议都会在未占用且编号 最小 的会议室举办。 如果没有可用的会议室，会议将会延期，直到存在空闲的会议室。延期会议的持续时间和原会议持续时间 相同 。 当会议室处于未占用状态时，将会优先提供给原 开始 时间更早的会议。 返回举办最多次会议的房间 编号 。如果存在多个房间满足此条件，则返回编号 最小 的房间。 半闭区间 [a, b) 是 a 和 b 之间的区间，包括 a 但 不包括 b 。

示例 1：

示例 2：

提示：

  • 1 <= n <= 100
  • 1 <= meetings.length <= 1e5
  • meetings[i].length == 2
  • 0 <= starti < endi <= 5e5
  • starti 的所有值 互不相同

算法分析

  • 这道题用两个堆维护即可，堆中放的元素是room对象
  • 第一个堆维护的是，会议室可用的时间以及会议室编号，是一个小根堆，排序规则是，可用时间从小到大，可用时间相同的时候，编号从小到大
  • 第二个堆维护的是，后续把第一个堆中符合某种条件的元素扔进来，按照会议室的编号从小到大即可
  • 具体算法过程如下：
    • 因为分配是根据会议起始时间来的，所以首先按照起始时间对会议进行排序
    • 将所有的会议室放入第一个堆(roomHeap)中
    • 以此弹出每一个会议室，表示当前可能分配的会议室
    • 如果发现最近的可用时间的会议室依然空闲，我们要把所有空闲的会议室都放到freeRoomHeap中，选择一个编号最小的，更新其下次可用的时间以此使用次数，并且把freeRoomHeap中的所有元素倒回
    • 如果发现没有空闲的，那么我们就选择当前弹出的（因为可用时间点最近，就算下面还有可用时间点一样近的，这一个会议室的编号也是最小的），更新其下次可用的时间以此使用次数
  • 这样做81个点过了80个，匪夷所思
